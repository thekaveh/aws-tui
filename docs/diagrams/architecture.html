aws-tui Architecture

Layered MVVM: View · ViewModel · Domain, wired by a composition root

View

  • • Textual widgets, panes, `.tcss` themes
  • • Never touches boto3 / aioboto3 / botocore
  • • Reads VM properties, calls relay-command execute(...)
  • • Subscribes to MessageHub via HubSubscriberMixin

ViewModel

  • • VMx VMs with reactive commands
  • • RootVM, file_manager VMs, emr_serverless VMs
  • • Never imports Textual; tests run headless
  • • construct -> run -> destruct -> dispose

Domain

  • • Service protocol: s3, emr-serverless plugins
  • • FileSystemProvider: LocalFS, S3FS, cross-FS engine
  • • AwsSession, ConfigStore, ThemeStore, KeychainBackend
  • • Only layer touching OS / AWS APIs / filesystem

aws-tui • layered MVVM • composition.py + app.py are the only trusted composition roots
