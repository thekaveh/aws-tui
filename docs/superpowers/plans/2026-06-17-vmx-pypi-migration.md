# 1. VMx Submodule → PyPI Dependency Migration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the `vendor/vmx` git submodule + `[tool.uv.sources]` path dependency with the published PyPI `vmx` package, leaving aws-tui consuming VMx exactly the same way at runtime but with a smaller checkout, no submodule cognitive overhead, and the v1.0 PyPI-release blocker lifted.

**Architecture:** One feature branch off `main`, seven tightly-scoped commits, single PR. The swap itself is one line of `pyproject.toml`; the rest is removing every trace of the submodule from CI / scripts / docs / memory and verifying every gate still passes against the PyPI wheel. The full test suite (522 default-tier + 40 snapshots + 9 MinIO opt-in, plus ruff + mypy strict + `scripts/check-layers.sh`) runs after every code-touching task — that's the test of correctness for an infra migration. Nothing in `src/` should change.

**Tech Stack:** Python 3.11+, uv-managed dependencies, `pyproject.toml` `[project].dependencies`, `[tool.uv.sources]` (currently routes `vmx` to the vendored path), GitHub Actions checkout with `submodules: recursive`, `scripts/bootstrap.sh` (submodule init), and prose docs that reference `vendor/vmx/`.

## 1.1. Global Constraints

- **Version pin:** `vmx>=2.4.0,<3.0.0`. Lower bound matches the vendored `python-v2.4.0` tag so API parity is guaranteed; upper bound caps before the next major. Adjust only if the user explicitly approves a different bound.
- **Branch protection:** `main` is PR-only. All work lands on `chore/vmx-pypi-migration` and merges via `gh pr create` + `gh pr merge --squash --delete-branch` (only after explicit user approval — don't auto-merge).
- **Never skip git hooks** (`--no-verify`, `--no-gpg-sign`, etc.). If a pre-commit hook fails, fix the underlying issue and re-commit.
- **No `src/` changes.** This migration must not touch `src/aws_tui/**`. If a step thinks it does, stop and re-read the plan.
- **Every code-touching task must end with all gates green.** Gates are:
  - `uv run pytest -q` → `522 passed, 9 deselected` + `40 snapshots passed`
  - `uv run ruff check .` → `All checks passed!`
  - `uv run mypy src/aws_tui` → `Success: no issues found in 70 source files`
  - `bash scripts/check-layers.sh` → `layer rules clean`
- **Per-task commits.** Don't batch tasks into one commit; the boundaries matter for review.
- **Co-author tag:** every commit ends with `Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>` (project convention — see `git log` on `main`).
